## Summary

Delivered the **Domains** module ‚Äî manual domain-registration tracking per client (no live registrar API in
v1), built on the Phase 1‚Äì5 tenant-isolation, RBAC, activity-log and Flux-CRUD foundation:

- **Domains** (`/admin/domains`) ‚Äî `Domain` records owned by a **required** client, carrying `domain_name`,
  registrar (free text), status, the key dates `registered_at`/`expires_at`, manual renewal logging
  (`last_renewed_at`, `renewal_cost` + `currency`), four nameservers (NS1‚ÄìNS4) and free-text WHOIS notes.
  Flux-table list with search (domain / client / registrar), status filter, expiry filter
  (expiring ‚â§30d / expired), sortable columns and pagination; create/edit/delete in modals.
- **Dedicated "Renew" action** ‚Äî a per-row quick flow that stamps `last_renewed_at` to today, sets the new
  `expires_at` (pre-filled one year past the current expiry, or one year from today if already expired),
  records the renewal cost, and flips an **Expired** domain back to **Active**. Chosen over edit-only date
  changes so manual renewal is a first-class action (idea.md ¬ß5 "manual renewal logging").
- **Expiry urgency** ‚Äî `isExpired()` / `daysUntilExpiry()` / `urgencyColor()` (red ‚â§7d or expired,
  yellow ‚â§30d, green else, zinc when no expiry), mirroring `ClientService`. `expires_at` is indexed for the
  Phase 7 reminder scan.
- **Status is manual in v1** ‚Äî admins set the status; the colour-coded urgency badge signals expiry visually.
  Auto-expiry (flipping to **Expired** once `expires_at` passes) is deferred to the Phase 7 scheduled command.
- **Enum** ‚Äî `DomainStatus` (Active / Expired / Pending Transfer / Cancelled; `label()` + `color()`).
- **RBAC** ‚Äî all actions gate on the already-seeded `domains.*` permission set via `DomainPolicy`; the renew
  action authorizes through `update`.
- **Client profile + sidebar** ‚Äî the profile gains a Domains stat card (the stat grid is now 4-up) and a
  Domains table card; the sidebar **Domains** item is enabled, gated on `domains.view`.

## Discovery Notes / Deviations

- **`domains.*` already seeded.** `RolesAndPermissionsSeeder::PERMISSIONS` already contains
  `domains.view/create/update/delete`, granted to `manager` (all) and `read-only` (`domains.view`). No role
  re-seed was needed. `billing` and `support` get no domain access by design (existing catalog) ‚Äî flagged,
  left as-is.
- **`DomainPolicy` auto-discovered.** Policies in this app are resolved by Laravel's `Model ‚Üí Policy`
  convention (no explicit `Gate::policy` registration), so the new policy needed no provider change ‚Äî same
  as every sibling module.
- **No separate "renewal" record.** Manual renewal updates the domain row in place (stamping
  `last_renewed_at`/`expires_at`/`renewal_cost`) rather than introducing a renewal-history table; the
  activity log captures the change. A dedicated renewal-history table can be added later if reporting needs it.
- **`company_id` not in `$fillable`** (matching the Phase 2‚Äì5 convention). It is stamped by the
  `BelongsToCompany` creating hook under auth, or set explicitly by the factory/seeder.
- **Client profile stat grid widened** from 3 to 4 columns to fit the new Domains card (now
  `grid-cols-2 lg:grid-cols-4`) ‚Äî the only layout change to an existing screen.
- No sub-agents were used; the codebase was small enough to inspect directly.
