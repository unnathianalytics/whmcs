# Phase 6 — Domains (Plan)

**Date:** 2026-06-24
**Status:** 📝 Planned — awaiting approval

## Goal

Deliver the **Domains** module from `docs/idea.md` §5: track domain registrations per client (no live
registrar API in v1). Each domain carries `registered_at` and `expires_at` (feeding the Phase 7 reminder
system), a status, registrar, nameservers (NS1–NS4), manual renewal logging, and a free-text WHOIS notes
field. The module reuses the Phase 1–5 tenant-isolation, RBAC, activity-log, and Flux-CRUD conventions, and
turns the **Domains** placeholders (sidebar item + client-profile card) into real, working features.

## Data Model — `domains` table

| Column | Type | Notes |
|---|---|---|
| `id` | bigint pk | |
| `company_id` | fk → companies, cascadeOnDelete | tenant isolation |
| `client_id` | fk → clients, cascadeOnDelete | required owner |
| `domain_name` | string, indexed | e.g. `example.com` |
| `registrar` | string nullable | free text (no API) |
| `status` | string default `active` | `DomainStatus` |
| `registered_at` | date nullable | start date |
| `expires_at` | date nullable, indexed | feeds reminders + urgency badge |
| `last_renewed_at` | date nullable | manual renewal logging |
| `renewal_cost` | decimal(10,2) nullable | manual renewal cost |
| `currency` | string(3) default `INR` | for renewal cost |
| `ns1`–`ns4` | string nullable | nameservers |
| `whois_notes` | text nullable | free-text |
| timestamps + softDeletes | | matches sibling modules |

Indexes: `status`, `expires_at`, `client_id` (queue/urgency filters + reminder scans).

## Implementation Steps

1. **Enum** — `DomainStatus` (Active/Expired/PendingTransfer/Cancelled) with `label()` + `color()`
   (green/red/yellow/zinc), patterned on `ServiceStatus`.
2. **Migration** — create `domains` per the table above (`php artisan make:migration`).
3. **Model** — `Domain` with `$fillable` (excluding `company_id`, per convention), `casts()` (status enum,
   dates, `renewal_cost` decimal:2), `client()` BelongsTo, the three expiry helpers, and
   `getActivitylogOptions()`.
4. **Relations** — `Client::domains()` (latest by `expires_at`) and `Company::domains()`.
5. **Policy** — `DomainPolicy` gating on `domains.*` (copy `TicketPolicy` shape).
6. **Factory + Seeder** — `DomainFactory` (inherit client's `company_id` like `ClientServiceFactory`,
   realistic registrar/NS/dates, `expiringInDays()` state); `DomainSeeder` creates 5 domains for the demo
   company; wire into `DatabaseSeeder`.
7. **Livewire Index** — list with search (domain name / client / registrar), status filter, expiry filter
   (expiring ≤30d / expired), sortable columns, paginated; create/edit modal (client required, domain_name
   required, status, registrar, dates, NS1–4, renewal cost/currency, whois notes); delete modal. Mirror
   `Admin/Services/Index`. Add a **"Renew" quick action** that stamps `last_renewed_at = today`, optionally
   advances `expires_at` by a year, and logs renewal — implemented as a small modal (manual renewal logging
   per idea.md). *(If this adds scope risk, fall back to editing dates via the edit modal and note it.)*
8. **Blade view** — Flux table + modals, expiry urgency badge via `urgencyColor()`, status badge.
9. **Route** — `Route::livewire('domains', AdminDomains::class)->name('domains')`.
10. **Sidebar** — replace the disabled Domains item with an active, `domains.view`-gated link (with the
    disabled fallback for users without the permission, matching the Tickets pattern).
11. **Client profile** — `domains()` computed + a Domains stat card and Domains table card on the profile.
12. **Tests** — feature coverage (see below). Run `php artisan test --compact --filter=Domains`.
13. **Pint** — `vendor/bin/pint --dirty --format agent`.

## Verification / Testing Plan

`tests/Feature/DomainsTest.php` (Pest), covering:
- **Tenant isolation:** a domain in company B is invisible to a company-A admin; `company_id` auto-stamped on
  create under auth.
- **Permission gates:** `viewAny`/`create`/`update`/`delete` enforced (a user without the permission 403s).
- **Create/edit:** validation (client + domain_name required; date ordering `expires_at >= registered_at`;
  currency size 3); persists correctly.
- **Expiry helpers:** `isExpired()` / `daysUntilExpiry()` / `urgencyColor()` thresholds (red ≤7/expired,
  yellow ≤30, green else, zinc when no expiry).
- **Filters:** status filter, "expiring" and "expired" expiry filters return the right rows.
- **Renewal action** (if implemented): stamps `last_renewed_at`, advances `expires_at`, persists.
- **Soft delete:** delete hides the row; scoped lookup respected.
- **Client profile:** `domains()` computed returns only that client's domains.

Plus `php artisan migrate:fresh --seed` (clean run, 5 demo domains) and a manual browser pass by the user
(list/filter/search/sort, create→edit→renew→delete, client-profile Domains card, sidebar link).

## Risks & Assumptions

- **`domains.*` already seeded** — verified in `RolesAndPermissionsSeeder::PERMISSIONS`; no role re-seed.
  `billing`/`support` roles get no domain access (existing catalog decision) — flagged, left as-is.
- **No registrar API** (idea.md "out of scope") — registrar and WHOIS are free-text; renewal is manual.
- **Client link required** — consistent with the Phase 5 ticket decision; a domain always belongs to a client.
- **`company_id` not fillable** — stamped by `BelongsToCompany`/factory/seeder, per Phase 2–5 convention.
- **Renewal modal is the one genuinely new pattern** vs. a straight `ClientService` copy; scoped as optional
  with a documented fallback to keep risk low.
- **Reminder wiring is Phase 7** — this phase only ensures `expires_at` exists and is indexed; no
  `reminders:send` integration yet.
